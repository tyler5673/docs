This page describes LangSmith API versioning and support policy for endpoints. For step-by-step upgrade instructions, see the [migration guide](/langsmith/endpoint-migration) for the relevant endpoint.

## API versions

Your API version controls endpoint behavior: which paths you can call, which parameters you can send, and the structure of responses. Major releases include changes that are not backward-compatible with the previous version. Upgrading to a new major version may require updates to your code.


## Version and support policy

| Term | Definition | Implication |
| ---- | ---------- | ----------- |
| **Supported** | The current and previous major version. | API requests succeed. We provide regular support. New features are added to supported versions. |
| **Deprecated** | A version that is _not_ the latest. Once a new version is released, the previous version is marked **deprecated**. | You can still use deprecated versions until they're _sunset_. We encourage upgrading to the latest version to benefit from new features. New features are not added to deprecated versions. |
| **Sunset** | A version that can _no longer be used_. | Requests sent to this version fail on or after the sunset date. You must upgrade using the [migration guide](/langsmith/endpoint-migration). |

### Differences between deprecation and sunset

| Term | Deprecation | Sunset |
| ---- | ----------- | ------ |
| **API** | Endpoints for deprecated versions _still function_. You can use our SDKs, REST, and so on. | Endpoints for sunset versions _stop working_ after the sunset date. The API returns an error for sunset versions. |
| **Support** | We provide regular support for deprecated versions. | Sunset versions are _no longer supported_ after the sunset date. |

## Backward-compatible changes

We consider the following changes to be backward-compatible within a major version (e.g., v2.0 → v2.1):

- Adding new optional request parameters to existing endpoints.
- Adding new properties to existing API responses.
- Adding new endpoints or new optional query parameters.
- Changing the order of properties in responses.
- Changing the length or format of opaque strings without changing semantics.

Upgrading to a new minor or patch release within the same major version should not require code changes.

## Breaking changes

Major version upgrades may include:

- Removing or renaming endpoints.
- Adding required parameters.
- Renaming or removing request or response fields.
- Changing request or response structure
- Changing required query parameters or request body schema.

See the [migration guide](/langsmith/endpoint-migration) for the full list of breaking changes between v1 and v2.

## Upgrade your API version

Upgrading your API version affects:

- The endpoint paths you call (`/api/v1/...` vs `/v2/...`).
- The parameters you can send and the structure of request bodies.
- The structure of responses (field names, required query params, etc).


### View the versions your project is using

You can see which API version your project uses by:

1. **API request logs**: Review your application logs or API monitoring tools for the endpoint paths and version prefixes in your requests.
2. **Code review**: Search your codebase for versioned endpoint paths (`/api/v1/` or `/v2/`).
3. **SDK version**: If you use the LangSmith SDK, check its version and documentation for which API version it uses by default for each endpoint.

### Perform the upgrade

When your code is ready for the new version:

1. Read the [migration guide](/langsmith/endpoint-migration) for breaking changes and request/response updates.
2. Update your code to use the new endpoints, parameters, and response handling (see the [migration checklist](/langsmith/endpoint-migration#migration-checklist) in the guide).
3. Test your integration against the new version.
4. Deploy the updated integration and monitor for errors.

### Roll back

If you need to revert after upgrading, switch your requests back to the previous version's endpoints and request/response format. Rollback is possible only until the previous version is **sunset**; after the sunset date, those endpoints no longer work.

### Stay informed

We announce API and endpoint changes and sunset dates on the [LangSmith changelog](https://changelog.langchain.com/). Subscribe or check the changelog regularly so you can plan upgrades before a version is sunset.
